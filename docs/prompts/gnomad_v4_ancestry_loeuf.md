# Prompt: gnomAD v4 ancestry-stratified LOEUF analysis

## Context

I'm working on a project analyzing genetic constraint in the melanogenesis pathway for a PEQG 2026 conference poster. The repo is `melanogenesis-constraints/`.

**Current state:** We already have LOEUF from gnomAD v2.1.1 for 129 genes from the Raghunath et al. (2015) melanocyte signaling network. The existing constraint data is in `data/network_constraint_data.csv` (not in repo but generated by `analysis/melanogenesis_network_constraint_v2.ipynb`). The current data file used for Phase 1 analyses is `data/network_constraint_gtex.csv`, which has these columns:

```
gene, functional_category, disease_class, LOEUF, pLI, betweenness_centrality, tissue_breadth, max_tpm, median_tpm_all
```

The v2.1.1 LOEUF was computed on the full gnomAD cohort, which is ~77% European. We want to know: **does the constraint architecture (pigment-specific genes = tolerant, signaling genes = constrained) hold up in non-European ancestry data, or is it inflated by European-specific variation?**

## Scientific motivation (for interpretation text)
The v2.1.1 LOEUF is ~77% European. If our key finding — that pigment-specific genes are uniquely LoF-tolerant — holds in African-ancestry data too, that strengthens the argument that the tolerance reflects genuine haploinsufficiency biology (these genes are recessive, so heterozygous LoF has no fitness cost in any population). If MC1R's LOEUF drops substantially in African data, that reveals the European estimate is inflated by population-specific functional variants and the "tissue breadth outlier" story needs nuance.
